# work_bridge v3 — 워크→카톡 중계 견고화 설계 (2026-06-03)

## 배경 / 문제
v2(`cycle_once_v2`)는 워크 안읽음방을 열어 본문을 읽고 워크 네이티브 메시지만 카톡으로
중계한다. 하지만 다음 문제로 신뢰성이 낮았다.

1. **모든 메시지가 미러로 오판** — `[발신자][시각]내용` 재조립 문자열에 미러헤더 정규식을
   걸어, 발신자+시각이 있는 *모든* 메시지가 걸려 0건 송신. (→ 커밋 69f8d24 에서 수정)
2. **reorder race** — 뱃지 y(캡처 A)와 방이름/순서(read_room_list_state=캡처 B)를 **따로**
   떠서, 그 사이 목록이 재정렬되면 엉뚱한 방을 열었다. 특히 처리에 수 초 걸리는 동안
   새 미러가 도착해 방들이 상단으로 튀면, 뒤쪽 후보일수록 한 칸씩 밀려 오방 오픈.
3. **미러 vs 워크네이티브 구분 부재** — Vision 이 미러 "[원발신자][시각]" 접두사를
   sender/time 으로 분리 추출하므로 content 기준 미러필터가 무력. 거래처 메시지가
   카톡으로 에코될 위험.
4. **안읽음 카운트 오염** — Vision 이 말풍선 옆 "7"(안읽음 수)을 content 에 포함 → "테스트 7".

## v3 핵심 설계 (cycle_once_v3)

### ① 인덱스 버리고 "재캡처 → 최상단 뱃지" 루프
- 안읽음 방은 카카오워크에서 **항상 목록 상단**으로 올라온다.
- 매 방마다 룸목록을 **새로** 캡처하고, `detect_blue_badge_rows`(순수 픽셀, Vision 아님,
  ~ms)로 **최상단 뱃지** y 만 뽑아 그 자리를 클릭한다.
- 캡처→클릭 간격이 ~300ms 라 그 사이 재정렬이 거의 없다. 인덱스·행높이 추정에 의존하지 않음.
- 방을 열면(=읽음) 뱃지가 사라져 **다음 안읽음 방이 top 으로** → 반복하면 자연 소진.
- 한 사이클 내 같은 제목 재오픈 시 `seen_titles` 로 무한루프 차단.

### ② 방제목 = 신뢰 원천
- 클릭으로 연 방의 **상단 제목**(`kakaowork_room_title` 영역, Vision)을 읽어
  그 제목으로 `_resolve_kakao_room` 매핑. "어디를 열었다고 가정"이 아니라 "실제로 연 방"이
  기준이므로 오방 송신이 원천 차단된다. (검증: row0/1/2 제목 3/3 정확)

### ③ 미러 식별 = 발신자 화이트리스트 `_is_mirror_origin`
- 워크 실멤버(`data/kakaowork_users.json`: 임재용/강현우/네노바) 발신만 워크 네이티브로 인정.
- 미러는 봇이 "[원발신자][시각]본문" 으로 올린 걸 Vision 이 분리 추출 → sender 가 카톡쪽
  사람(농장/거래처) → 비멤버 → 제외. 발신자 불명도 안전하게 제외.
- 재전송 방지는 `work_bridge_sent_ledger.json`(md5(room|sender|time|content)).

## 실행
```
python main.py work-bridge --v3 --once         # 1사이클 검증
python main.py work-bridge --v3                 # 데몬(기본 20s, _STOP 로 종료)
python main.py work-bridge --v3 --dry-run       # 송신 없이 분류만
```

## 검증 현황 (2026-06-03)
- 컴포넌트 전부 검증: 뱃지감지 / 뱃지y클릭 오픈 / 제목식별(3/3) / 제목→카톡매핑 /
  미러·네이티브 분류 / 송신+카톡 도착(강현우 "테스트" → 카톡 '네노바 영업' 노란말풍선).
- 0 뱃지일 때 정상 무동작 확인.
- title-verify 가 인덱스기반 오방클릭 2건을 실제로 차단(설계 동기 입증).

## 알려진 한계 / TODO
- **이 PC 자체 입력은 뱃지가 안 생긴다** — 이 PC(네노바 계정)가 워크에 직접 친 글은
  읽음 상태라 뱃지 없음 → v3 가 못 봄. 단, 실사용은 "타인이 워크에 올림"이 대부분이고
  그건 이 PC에 안읽음(뱃지)으로 잡힌다. 자체입력까지 잡으려면 전 방 주기 스캔 필요(고비용).

  되돌아오는 edge case 는 ledger 로 1회만 막힌다. 더 견고한 시간 워터마크는 후속 과제.
- 카톡 분리창 미존재 시 검색→오픈에 최대 10s. send_message_to_room 은 '정확한 제목 분리창'
  없으면 실패 반환(오방 송신 방지).
