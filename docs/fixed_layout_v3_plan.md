# 고정 레이아웃 통합 운영 기획서 (v3)

> 작성 2026-06-03. 그동안의 불안정(검은화면/11→3방/방못엶/위치변동)의 **근본 원인**은
> 앱 위치가 매번 흔들리고 캡처 영역이 유동적이었던 것. → **두 앱을 고정 배치 +
> 캡처영역 고정(학습) + PrintWindow 캡처**로 안정성을 근본 확보한다.

## 1. 사용자가 정의한 운영 = 3기능 동시

한 시스템이 대기하다가 새 게 생기면 처리:
1. **카톡→워크**: 카톡 새 메시지(안읽음) → 워크 미러방 송신
2. **워크→카톡**: 워크 새 메시지(남이 보낸 것) → 카톡 원본방 송신
3. **저장·분석**: 모든 대화 내용을 `collected_data.jsonl` 누적 → 이슈트래커/CTI 분석

## 2. 핵심 아이디어 (사용자 제안) — 고정 레이아웃 + 캡처영역 학습

```
화면 왼쪽:  카톡 앱  — 고정 위치(x,y) + 고정 가로폭(w) + 높이(h)
화면 오른쪽: 워크 앱  — 고정 위치(x,y) + 고정 가로폭(w) + 높이(h)
  두 앱을 동시에 띄워 서로 안 가림.

각 앱의 캡처 영역을 1회 "학습"(설정)해 좌표 고정:
  - 카톡: 룸목록영역 / (분리창 대신) 방 행 좌표
  - 워크: 룸목록영역 / 대화창(우측패널)영역
  → PrintWindow 로 그 고정 영역만 캡처 → Vision/픽셀 분석 → 클릭/입력 좌표도 고정
```

효과:
- **위치 흔들림 0** → 캡처/클릭 좌표 항상 동일 (그동안 사고의 근본 제거)
- **두 앱 동시 표시** → 카톡·워크 번갈아 작업해도 경합 없음
- **PrintWindow** → 가려져도/백그라운드도 캡처(검은화면·포커스강탈 불필요)
- 한 번 학습하면 매번 재사용

## 3. 레이아웃 설정 (window_positions.json 확장)

```json
{
  "kakaotalk_main": {"x":0,   "y":0, "w":500, "h":1000},
  "kakaowork_main": {"x":520, "y":0, "w":520, "h":1000},
  "capture_regions": {
    "kakaotalk_roomlist": {"dx":..,"dy":..,"w":..,"h":..},   // 앱 내부 상대좌표
    "kakaowork_roomlist": {"dx":..,"dy":..,"w":..,"h":..},
    "kakaowork_chatpanel":{"dx":..,"dy":..,"w":..,"h":..}
  }
}
```
- 앱 위치·크기는 `lock_*_window` 로 매 시작 시 강제 고정.
- 캡처영역은 앱 좌상단 기준 **상대좌표(dx,dy,w,h)** → 앱 위치 바뀌어도 자동 보정.

## 4. 학습(설정) 도구 — `main.py layout-setup`

1회 실행해 좌표 확정:
1. 카톡/워크를 고정 위치로 이동(lock)
2. 각 앱 PrintWindow 전체 캡처 → 사용자가 영역 확인(또는 Vision 으로 룸목록/대화창
   경계 자동 감지) → capture_regions 저장
3. 검증: 저장 영역만 캡처해 화면에 보여줘 OK 받기
- "학습" = 이 좌표 1회 확정. 이후 운영은 이 값 재사용(재학습 불필요).

## 5. 통합 운영 루프 — `main.py monitor`(확장) 또는 신규 `run-bidir`

```
초기: lock_kakaotalk_window + lock_kakaowork_window (고정 배치)
매 사이클:
  [A] 카톡→워크 (기존 모니터 로직, 뱃지모드)
      - 카톡 룸목록 PrintWindow 캡처 → 안읽음 방 → 열어서 저장+미러
      - collected_data.jsonl 누적  [기능3]
  [B] 워크→카톡
      - 워크 룸목록 PrintWindow 캡처(고정영역) → unread 증가 방 탐지
      - 그 방 열기(고정 좌표 클릭 — Ctrl+K 대신 룸목록 행 직접 클릭)
        → 대화창 PrintWindow 캡처 → 본문 추출
        → 미러본/봇 제외 → 새 내용만 카톡 원본방 송신
  → 한 프로세스가 순차 처리 → 화면경합·락 불필요(단일 소유자)
```

핵심 변경:
- **단일 프로세스**로 통합 → 카톡 락 경쟁 제거(현재 모니터+work_bridge 2프로세스 문제 해소)
- **방 열기 = 고정 좌표 클릭** (Ctrl+K 안 되던 문제 우회 — 룸목록에서 해당 행 좌표 클릭)
- **모든 캡처 = PrintWindow 고정영역**

## 5.5 워크→카톡 판별 — "마지막 미러 시각" 기준 (사용자 제안, 핵심)

복잡한 정규식 에코필터 대신 **시간(timestamp) 기준**이 가장 깔끔·정확.

### 1차: 시간 워터마크 (주 판별)
```
모니터가 카톡→워크 미러할 때, 방별 '마지막 미러 시각'을 기록.
  work_mirror_watermark[방] = 마지막으로 미러한 메시지의 시각(오후 3:50 등)

워크방 대화창 본문에서:
  - 워터마크 시각 이후의 메시지 = 워크에만 있는 내용(카톡엔 없음) → 카톡 송신
  - 워터마크 이전/같음 = 카톡것(미러) → 무시
```

예: 워터마크 3:50. 워크방에 `[심동욱] 3:48 ...`(무시) / `3:55 수국 2박스`(→카톡 송신).

장점(텍스트 비교 대비):
- 내용 무관 — 같은 문구 반복돼도 안 헷갈림
- OCR 흔들려도 **시각만 맞으면** 판별됨
- 카톡 `extract_delta`(시간순 누적)와 자연스럽게 호환

### 2차: 텍스트 기준점 (보조 — 같은 분(分) 애매함 보정)
- 카톡 시각은 **분 단위**라 같은 분에 미러것+워크것이 섞이면 모호.
  → 같은 분 메시지는 '마지막 미러 메시지 텍스트'와 일치하는 것까지는 무시,
    그 뒤는 신규로(텍스트 기준점 병행).
- `_looks_like_mirror_header`([발신자][시각] 형식) = 미러본 추가 안전망.
- 전송분은 ledger 기록 → 재전송 방지.

### 구현
- 모니터 미러 송신부에서 `work_mirror_watermark` 갱신(방별 마지막 미러 시각·텍스트).
- 워크→카톡 단계: 본문 메시지의 time 을 분 단위 비교 → 워터마크 이후만,
  같은 분은 텍스트 기준점으로 보정 → 카톡 원본방 순서대로 전송(여러 개 가능).
- 워크도 **안읽음순 정렬** → 안읽음 뱃지 방만 검사(카톡 방식과 동일).

## 6. 워크→카톡 방 열기 — Ctrl+K 대신 좌표 클릭

현재 막힌 Ctrl+K 대신:
- 워크 룸목록(고정영역) 캡처 → Vision 으로 "새 메시지 방"이 몇 번째 행인지 확인
- 그 행의 **고정 좌표(앱위치+행오프셋) 클릭** → 방 열림 → 대화창 캡처
- 좌표가 고정이라 클릭 안정. (카톡 모니터의 행클릭과 동일 원리)

## 7. 단계별 구현

- **P1 레이아웃 설정**: `lock_kakaowork_window` 추가 + window_positions.json 에
  kakaowork_main/capture_regions 추가 + `layout-setup` 도구
- **P2 고정영역 캡처**: PrintWindow + capture_regions 로 카톡/워크 룸목록·대화창 캡처 헬퍼
- **P3 워크방 좌표클릭 열기**: 룸목록 캡처→행 식별→고정좌표 클릭→대화창 캡처
- **P4 통합 루프**: 한 프로세스에 [A][B][기능3] 통합, 락/2프로세스 제거
- **P5 E2E + 안정화**: 라이브 검증, 좌표 미세조정

## 8. 리스크·고려

- 화면 해상도/배율 바뀌면 고정좌표 재학습 필요 → layout-setup 재실행으로 대응
- 워크 룸목록 스크롤(방 많으면) — 상위 N개만 보이므로 unread 방이 상위에 오는지 확인
  (카톡처럼 새 메시지가 위로 올라오면 OK)
- 크레딧: 캡처는 무료(PrintWindow), Vision 분석만 크레딧 — 새 방 있을 때만 호출
- 단일 프로세스 = 한 기능이 느리면(예: 사진 다운로드) 다른 기능 지연 → 타임박스 필요
